Common Website Layout Elements

There are many global layout elements that are shared by most websites, regardless of their content or design. Which ones can you think of?

  • Masthead
  • Logo
  • Tagline
  • Primary Navigation
  • Primary Content
  • Secondary Content (Sidebar)
  • Footer

Common Website Element IDs

Since these elements tend to appear on most websites, it makes sense to adopt some conventions for how we write our base HTML so that we can re-use some of it from site to site.

<div> tags are commonly used to group these global layout areas together. Since there is only one of each of these elements per page (1 masthead, 1 footer, etc.) we use unique id attributes to identify the different areas. These ids can be used in our CSS later when we construct the layout.

For consistency and code re-use purposes, it is a good idea to decide on some generic IDs to use over and over across your projects. Some commonly used IDs are:

  • masthead
  • logo
  • tagline
  • navigation/nav/nav-primary
  • content/content-primary
  • sidebar/content-secondary
  • footer/site-info

Re-usable HTML Skeleton

We can combine the common elements/ids we just discussed to create a generic HTML skeleton that can be used for most HTML layouts. To begin, we create a div to hold the entire site, which is often given the id "container", "wrapper" or "bucket".

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
	<head>
		<title>HTML Skeleton</title>
	</head>
	<body>
		<div id="container">
		
		</div>
	</body>
</html>

HTML Comments

One good habit is to include an HTML comment with the div ID right after the closing div. This helps you to find extra/missing closing DIVs when your layout gets more complex. An HTML comment takes the following form:

<!-- YOUR COMMENT TEXT -->

Anything within a comment will not be shown to the user. Here is the container example with the extra comment.

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
	<head>
		<title>HTML Skeleton</title>
	</head>
	<body>
		<div id="container">
		
		</div><!-- #container -->
	</body>
</html>

HTML Skeleton: Masthead

Every website includes a masthead area with a logo that links back to the home page. Lets add this to our template.

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
	<head>
		<title>HTML Skeleton</title>
	</head>
	<body>
		<div id="container">	
			
			<div id="masthead">				
				<div id="logo"><a href="/">Company Name</a></div>
				<div id="tagline">Tagline</div>
			</div><!-- #masthead -->
			
		</div><!-- #container -->
	</body>
</html>

Note: The link (anchor tag) back to the home page has an href (the URL to link to) attribute of "/". This is a shortcut for taking the user back to the root/home page of the site. We would also end up replacing the text "Company Name" with an <img> tag of the company logo in most cases.

			<div id="masthead">

				<div id="tagline">Tagline</div>
			</div><!-- #masthead -->
			
			<div id="navigation">				
				<ul>
					<li><a href="/">Home</a></li>
				</ul>
			</div><!-- #navigation -->
			
		</div><!-- #container -->

For our template we will only include the home link, since the actual navigation links will vary from site to site. For each new navigation link we will just add another list item to the unordered list.

CSS for Layout

By default, the HTML skeleton we just created is vertically oriented. Since the div tags are display: block by default, they simply take up 100% of the available width and include line breaks beneath them. Most websites do not lay out this way (although it is common for mobile versions). What we need is a way to go from this vertical stacking orientation to a horizontal stacking orientation. This is where floats come in.

It is helpful to think of floats in terms of images in magazine layout, as that is the original problem they were created to solve. In magazine/newspaper layouts, images commonly float on the right or left of the column, with text flowing automatically around them.

The image to the right is an example of a floated image. This text will flow around the image automatically, occupying any available space. Floats are still used for this purpose, whenever you want text to wrap around an image (commonly in employee bios, product detail pages, etc.). The behavior of the image to the right comes from the following CSS: 

img {
	float: right
}

CSS: Floats

The float an image, you can use the float CSS property.

Values

  • left Float the element to the left. Text will flow around it on the right.
    float: left
  • right Float the element to the right. Text will flow around it on the left.
    float: right
  • none Used to remove floating behavior from an element.
    float: none

Floats for Layout

But floats can do so much more than that. In fact, floats are the primary approach to creating column layotus in CSS. Floats can be applied to any HTML element (not just images). By applying floats to div tags, we can create multi column layouts. This is because 2 elements that are floated will appear side by side. Note Borders are just for clarity.

CSS

div {
	float: left;
}

HTML

<div>Div A</div>
<div>Div B</div>

Floats & Width: The Shrink Wrap Effect

In the previous example, you probably noticed that the div tags no longer take up the 100% width they normally would. Floating any element causes it to "shrink wrap" to the size of its content. So if we increase the size of the content in those divs, the width will increase.

For this reason, you almost always declare a width when you float div tags for layout purposes. This allows you to specify exactly how wide they should be and the content will wrap as necessary.

Example

Div A has a lot more content and gets wide
Div B

Floats & Width: The Shrink Wrap Effect

Updated CSS

#sidebar {
	float: left;
	width: 300px;
}

#content {
	float: left;
	width: 800px;
}

Updated HTML

<div id="sidebar">Div A has lots of content but now has a width</div>
<div id="content">Div B</div>

Example

Div B

A Problem Arises

What happens when we add our footer into the previous example?

Updated HTML

<div id="sidebar">Sidebar has lots of content but now has a width</div>
<div id="content">Content Div</div>
<div id="footer">Footer Div</div>

Example

Footer

As you can see, the footer is wrapping around the float. This is what we would expect, but its not what we want.

The Solution: Clearing Floats

Whenever we no longer want our HTML tags to wrap around our floated elements, we just use the clear CSS property to clear the float. Values for the clear property

  • left Clear elements that are floating on the left.
    clear: left
  • right Clear elements that are floating to the right. 
    clear: right
  • none Remove clearing behavior from an element.
    clear: none

Clearing the Footer

Lets add clear: left to our footer and see what happens

Updated CSS

#sidebar {
	float: left;
	width: 300px;
}

#content {
	float: left;
	width: 800px;
}

#footer {
	clear: left;
	width: 1100px;
}

Clearing the Footer

Problem solved! The clear: left property on the footer clears the floats from the sidebar and content div so the footer lays out the way we intend.

Result

Sidebar has lots of content but now has a width

Footer

Float Order

If we use float: right on our sidebar and content divs instead of float left, we can get the sidebar on the right side. We also need to change the footer to clear: right.

Updated CSS

#sidebar {
	float: right;
	width: 300px;
}

#content {
	float: right;
	width: 800px;
}

#footer {
	clear: right;
	width: 1100px;
}

Float Order: Result

Using float: right instead of left, the sidebar ends up on the right side instead of the left.

Float: Summary

Using a combination of the float and clear CSS properties, you can create multi-column layouts in HTML & CSS.

When you are using floats to create multi-column layouts, declare a width on your div tags to avoid the shrink wrap effect, where the width of the div is determine by the amount of content you put in it.

If you want a div to no longer float as one of the columns in your layout, use the clear property to stop the floating behavior. If you are using left floats you will declare clear: left. If you are using right floats you will use clear: right. You can also use clear: both to clear all floats.

Often times you want to be able to float images to the right or left in your content area. It is common to declare some re-usable classes in your CSS for this purposes. These classes usually include some margin to prevent the text that wraps around the image from butting up against it.

.image-right {
	float: right;
	margin-left: 20px;
	margin-bottom: 20px;
}

.image-left {
	float: left;
	margin-right: 20px;
	margin-bottom: 20px;
}

Positioning

CSS includes another set of properties for layout that are more suited for pixel specific placement of elements. Positioning is not generally used for creating column layouts. Instead it is most useful for when you need to position an HTML element at a very specific location. The logo and tagline within a masthead are good examples of commonly positioned elements.

Positioning is done through a combination of the css position property and the css properties top, bottom, left and right

CSS: Position Property

The position property has 3 possible values:

  • absolute Position the element as a specific pixel or percentage location 
    position: absolute;
  • relative Move the element relative to its normal position. 
    position: relative;
  • static This is the default and is used for reseting other position values 
    position: static;

position: absolute

Absolute positioning is the most common of the three. In absolute positioning, we use a combination of the top, bottom, left and right properties to specify exactly where the element should be positioned. By default, elements are positioned relative to the body tag.

Place an element in the top left corner of the screen

position: absolute;
top: 0;
left: 0;

Place an element in the bottom right corner of the screen

position: absolute;
bottom: 0;
right: 0;

Place an element 50 pixels from the left edge of the screen and 100 pixels from the top

position: absolute;
left: 50px;
top: 100px;

position: absolute

It is rare that we want to position an element relative to the body tag (which is the default). Consider the example of a masthead div with a logo in it. If I want the logo to be 50 pixels from the top of the masthead and 100 pixels from the top, how do I achieve this? Here is what happens when you just add absolute positioning to the logo div (the logo div is in blue, the masthead is red)

CSS

#logo {
	position: absolute;
	left: 50px;
	top: 100px;
}

HTML

<div id="masthead">
	<div id="logo">My Logo</div>
</div>

position: absolute

Look for the "My Logo" text on this page. As you can see it is not inside of the red masthead div. It is being positioned based on the element it is inside of that has a position property set to either relative or absolute. In this case, the slide itself has a position of relative, so the logo div is placed 100 pixels from the top and 50 from the left.

Result

My Logo

position: relative

Lets add position: relative to the masthead div and see how that changes the results.

CSS

#masthead {
	position: relative;
}

#logo {
	position: absolute;
	left: 50px;
	top: 100px;
}

HTML

<div id="masthead">
	<div id="logo">My Logo</div>
</div>

position: relative

The "My Logo" div is now 100 pixels from the top of the masthead div and 50 pixels from the left. You probably noticed that the masthead div seems to have no height. This is because absolute positioned elements are removed from the normal flow of the document. In this example, we would want to add a height to the masthead to prop it open since its only element is absolutely positioned.

Result

My Logo

position: relative

Lets add a height to our masthead.

CSS

#masthead {
	position: relative;
	height: 200px;
}

#logo {
	position: absolute;
	left: 50px;
	top: 100px;
}

HTML

<div id="masthead">
	<div id="logo">My Logo</div>
</div>

position: relative

Now that the masthead has a height of 200 pixels declared, it will always be that tall, even if it has no children or its children are absolutely positioned. This is useful when you have a background image for your masthead you want to be completely visible regardless of what content you put into the masthead.

Result

My Logo

position: relative

Position relative can also be used to move elements based on where they would normally be placed. This is actually not a very common thing to need to do, lets look at why. Lets say we have 2 paragraphs and we position one of them relatively and set it to be 50 pixels from the top of where it normally would be.

CSS

#p1 {
	position: relative;
	top: 50px;
}

HTML

<p id="p1">Paragraph 1</p>
<p>Paragraph 2</p>

position: relative

What a mess! The first paragraph is overlapping the second paragraph. Remember how we said that absolutely and relative positioned elements are removed from the normal flow of the page? This means they can overlap each other. This can be helpful sometimes, but it can also cause these types of collisions. This is one of the reasons floats are used for layouts rather than postioning.

Paragraph 2

Positioning Summary

Positioning is used much less commonly than floating and is not generally used for creating layouts. It is very useful, however, when you need to place an HTML element in a very specific pixel location. In this case, you should consider what you want to position it inside of and give that parent element a position of relative. We looked at the most common usage of setting a masthead to be position relative and then setting a logo inside of it to be absolutely positioned within it.
