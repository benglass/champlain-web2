What is HTML?

HTML is a language for describing different types of content such as text, images, lists and videos.

This is a Paragraph

<p>The massive, frustrated energies of a mainly young, disillusioned electorate that has long since abandoned the idea that we all have a duty to vote. This is like being told you have a duty to buy a new car, but you have to choose immediately between a Ford and a Chevy.</p>

This is an Image

<p><img src="car.jpg" alt="A chevy" /></p>

This is a List

<h2>Groceries</h2>
<ul>
	<li>Milk</li>
	<li>Eggs</li>
	<li>Orange Juice</li>
</ul>

HTML is not for Design

In the early days of HTML, there were tags and attributes available for determining how the content should be displayed. Consider using the following HTML code for your page headings.

<font size="4" color="red" face="verdana">About Us</font>

What's the Problem?

This approach might work fine for a 1 page website, but what problems would occur as you add additional pages?

<font size="4" color="red" face="verdana">About Us</font>
  • Re-using styles requires us to copy the entire line of code
  • Changing all headings requires editing all pages individually

CSS is the Solution

Using CSS we can separate the design from the content, making it easy to change one without changing the other.

So instead of this...

<font size="4" color="red" face="verdana">About Us</font>

We can do this

<style>
h1 {
	font-size: 14px;
	color: red;
}
</style>

<h1>About Us</h1>

What is CSS?

CSS (Cascading Style Sheets) is a language for describing how HTML documents should be visually formatted in terms of typography, multi-column layout, decorative images, etc.

CSS can be placed into .css files and linked to HTML pages to allow their contents to be re-used on multiple pages of a website. This makes it easy to update the design of all pages on a website easily by editing a single file.

CSS Rules

The basic unit of CSS is called a rule.

A rule consists of a property and a value separated by a colon.

font-size: 14px

Multiple rules can be combined by placing a semicolon between them

font-size: 14px; line-height: 18px; color: red;

There are hundreds of CSS properties for changing everything from font size to color to adding background images and specifying the layout of a page. The following properties represent some of the more common ones.

CSS & HTML Together

There are three ways to use CSS with HTML:

  • Inline: CSS Rules are attached directly to the HTML tag
  • Embedded: CSS is placed in a separate section of the HTML document
  • Linked: CSS is placed in a separate .css files and linked to the HTML document

Inline CSS

CSS can be attached directly to an HTML tag using the style attribute. The value of the style attribute is set to a series of CSS rules.

<p style="color: red">This paragraph is now red</p>
<p style="font-weight: bold; color: red">This paragraph is now red AND bold</p>

Pros: Easy, clear

Cons: Not re-usable or maintainable (no better than the font tag example)

When to Use: Almost never

Embedded CSS

All CSS for a page can be placed between style tags (not attributes).

<style type="text/css">

p {
	font-weight: bold;
	color: red;
}

</style>

Pros: More re-usable & maintainable

Cons: Not re-usable across multiple HTML pages

When to Use: Styles that are only used by a single HTML page

Linked CSS

All CSS is placed a separate .css file (e.g. styles.css) and added to the HTML page using the link tag.

Example:

Contents of styles.css (note: no style tags)

p {
	font-weight: bold;
	color: red;
}

Inside the head tag of the the HTML Page

<link rel="stylesheet" type="text/css" href="styles.css" media="all" />

Linked CSS

Pros: Re-usable across multiple HTML pages, easiest maintenance, easiest validation, update 100+ page website by changing 1 file

Cons: ???

When to Use: Almost always

Example:

<link rel="stylesheet" type="text/css" href="styles.css" media="all" />

A New Problem

When the CSS was applied directly to our tags in the Inline technique, it was very clear to what element the CSS was being applied.

<p style="color: red">This paragraph is red</>

But with our CSS separated from our HTML using either the Embedding or Linking technique, how does the browser know which CSS rules should be applied to which HTML tags?

color: red
<p>This paragraph should be red</>

A New Solution: CSS Selectors

A CSS Selector describes which HTML tags the CSS rules should apply to. For example, to apply some CSS to all paragraph tags on a page, you would use the selector p followed by the CSS rules. CSS rules are grouped together using curly braces {}:

p {
	color: red;
}

In the example above, p is the selector and the CSS rule is that the color property should be set to red. The effect of this rule would be to make all text inside of paragraph tags red. This rule would apply to the following HTML:

<p>This paragraph would turn red</p>
<p>... and so would this one</>

CSS Selectors: The Tag Selector

The Tag Selector is used to select specific HTML tags (a, img, p, body, etc.) and apply CSS rules to all of them (all paragraphs, all images, etc.). It takes the form of the HTML code for the HTML tag without the < and > symbols.

Examples

p {
	color: red;
}

img {
	border: 1px solid #000000;
}

body {
	font-size: 16px;
}

CSS Selectors: The Class Selector (.)

The Class Selector selects all HTML tags that have a certain value for their class attribute. It takes the form of a period (.) followed immediately by the name of the class (no spaces). Any HTML tag can use the class attribute and multiple tags can share the same class (there can be more than one element with a class of "caption")

Example CSS

.caption {
	text-transform: lowercase;
}

Matching HTML

<p class="caption">This paragraph has a class of "caption" and therefore will be changed to lowercase</p>
<p>This paragraph would not match</>

CSS Selectors: The ID Selector (#)

The ID Selector selects all HTML tags that have a certain value for their id attribute. It takes the form of a pound symbol (#) followed immediately by the name of the id (no spaces). Any HTML tag can use the id attribute but each id must be unique (there can be only one tag with an id of "masthead")

Example CSS

#masthead {
	background-color: black;
}

Matching HTML

<div id="masthead">Paint it Black</div>

CSS Selectors: Combining Selectors

Selectors can be combined by placing a space between them. This represents a child relationship, meaning you are looking for HTML tags that match the second selector that are WITHIN other HTML tags that match the first selector.

Example CSS

#masthead a {
	text-decoration: underline;
}

Matching HTML

<div id="masthead">
	<a href="index.html">This link now has an underline</a>
</div>

Text Properties: Alignment

text-align: Control alignment of text.
Values: left, center, right, justify

Examples

text-align: center
text-align: right

Text Properties: Leading/Line Height

line-height: Amount of leading + unit. Same units as text.
Note: EMs are useful here.

Examples

line-height: 18px
line-height: 1.5em

Text Properties: Bold

font-weight: Keyword (normal, bold) OR numeric weight (200, 800) if font supported.

Examples

font-weight: bold
font-weight: 800
font-weight: normal

Text Properties: Italic

font-style: Keyword (normal, italic, oblique)

Examples

font-style: italic
font-style: normal

Text Properties: Underlines, Overlines, etc.

text-decoration: Keyword (none, underline, line-through, overline, blink)
Note: Useful for adding/removing underlines to links

Examples

text-decoration: none
text-decoration: underline

Text Properties: Changing Case

text-transform: Keyword (none, capitalize, uppercase, lowercase)
Note: Useful for heading styles

Examples

text-transform: uppercase
text-transform: lowercase
text-transform: none

Typefaces in CSS: Font Stacks

Since fonts on web pages are loaded from the user's computer, CSS includes a concept of a Font Stack, or a list of fonts to use in order from left to right. If the first font is not found, the second will be used and on down the line.

Fonts in a stack are separated by a comma and the last item in the list can be a generic font type to use if none of the fonts are found (serif, sans-serif, monospace). Any font names with spaces in them are enclosed in single quotes (e.g. 'Times New Roman'). Webfonts loaded from Google or other sources are declared in the same way as regular fonts once they have been added to the page.

Text Properties: Font Family

font-family: font1, font2, generic font

Examples

font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif
font-family: Georgia, Times, 'Times New Roman', serif
font-family: 'Palatino Linotype', 'Book Antiqua', serif

Units in CSS: Absolute Units

Whenever you are specifying a numeric value in CSS (font size, width, margins, etc.) you have to declare the value followed by the unit with no space in between. The only exception is 0, which requires no unit. There are two types of units: Absolute and Relative. Absolute units represent a specific numeric value (18 pixels, for example).

Points (pt)

Standard point sizing. For print stylesheets only.

font-size: 18pt

Pixels (px)

Size in pixels, the standard unit for web sites.

font-size: 18px

Units in CSS: Relative Units

Relative units are calculated based on the default value.
If the default is 18px and 200% is used, 36px is the result

Percent (%)

Based on the default size. 200% = double the size. 50% = half the size.

font-size: 200%

Em (em)

Relative, multiple by default font size. If font size is 12 and 1.5em is used, resulting font size is 18 (12 x 1.5)

font-size: 1.5em

Text Properties: Size

font-size: Size of the text + unit (no space between).
Valid units: Points (pt), Pixels (px), Percent (%), Ems (em).

font-size: 18px
font-size: 200%
font-size: 1.5em

Text Properties: Shorthand

CSS allows you to specify the font size, leading, family, style and weight all in one line with the shorthand font property.

font: style weight size/line-height family

Examples

font: italic bold 12px/18px Arial, sans-serif
font: 22px/1.5em Georgia, 'Times New Roman', serif
font: bold 14pt/20pt 'Helvetica Neue', sans-serif

Color in CSS

Whenever you are specifying color in CSS there are 3 methods: Hex Code, RGB Value, and Keyword. Hex Code is a 6 digit alphanumeric value (letters and numbers) preceeded by a pound symbol (#). It is easily retrieved in Photoshop and is the standard method in CSS. Keywords are rarely used as there is not a keyword for every color.

Example: The Color White

Keyword: white
Hex Code: #FFFFFF
RGB: rgb(255, 255, 255)

Text Properties: Color

color: Keyword OR Hex Code OR RGB

Examples

color: white
color: #F3F3F3
color: rgb(255, 255, 255)

Background Properties

Background images are a very important component in CSS. They are used for everything from bullets in lists to advanced column layouts. A background image is applied to an HTML element (like a paragraph) and you can specify what image to use, whether and how it should be tiled and where it should be positioned.

Background Properties: Color

background-color: Keyword (red, blue) OR Hex Code (#000000) OR RGB (rgb(0, 0, 0)). The transparent keyword can also be used.
Note: Hex codes are the standard format & easily copied from Photoshop

Examples

background-color: transparent
background-color: black
background-color: #000000
background-color: rgb(0, 0, 0)

Background Properties: Image

background-image: url(path/to/image.gif)
Note: This is relative to your CSS file

Examples

background-image: url(../images/smiley.png)
background-image: url(rounded-corners.png)
background-image: url(http://google.com/logo.png)

Background Properties: Repeat

background-repeat: repeat-x, repeat-y, repeat or no-repeat

Examples

background-repeat: no-repeat
background-repeat: repeat-y
background-repeat: repeat-X

Background Properties: Position

background-position: This property defines where the background is positioned within the HTML element. This is only relevant when the repeat is NOT set to repeat. The first value is the x position and the second is the y position. The two values are separated by a space.

These values can be specified either using keywords or numeric values. Keywords include: left, right, top, bottom, center. Numeric values must include units unless they are 0

Background Properties: Position

background-position: x y

Example: 50 pixels from the left, 100 pixels from the top

background-position: 50px 100px

Example: Centered

background-position: center center
background-position: 50% 50%

Example: Bottom Right

background-position: right bottom
background-position: 100% 100%

Background Properties: Shorthand

CSS allows you to specify the background color, image, repeat and position all in one line using the shorthand background property.

background: color url repeat position

Example

background: transparent url(../images/bg.png) no-repeat center center
background: #000000 url(rounded-corners.png) repeat-x 0 100%
background: red url(divider.png) repeat-y right top

List Properties: Type

list-style-type: Keyword (none, circle,upper-roman, etc.)

Examples

list-style-type: none
list-style-type: square
list-style-type: lower-alpha
